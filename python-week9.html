🌍 1. 什么是面向对象编程（OOP）

🔑 核心定义

面向对象编程（Object-Oriented Programming，OOP）是一种以"对象"为核心的编程范式。它把现实世界中的事物抽象为代码中的"对象"，每个对象拥有自己的数据（属性）和行为（方法），通过对象之间的交互来完成程序功能。

Python 是一门支持多范式的语言，但其 OOP 支持极为完整。Python 中一切皆对象——整数、字符串、函数本身都是对象！

📌 OOP 诞生背景：为什么需要它？

早期编程是"面向过程"的——把程序看成一系列步骤，一步一步执行（如C语言风格）。当程序规模增大，就暴露出严重问题：

  • 数据和函数分离，逻辑难以追踪维护
  • 全局变量泛滥，函数间耦合严重
  • 代码复用困难，改动一处影响多处
  • 难以对复杂的现实世界建模

OOP 把相关的数据和操作数据的函数打包在一起，形成更贴近真实世界的"对象"模型，大幅降低了大型程序的复杂度。

🌰 生活类比：汽车与设计图
  • 类（Class） = 汽车设计图纸 → 描述汽车有什么（品牌、颜色、速度）和能做什么（启动、加速、刹车）
  • 对象（Object） = 按图纸造出来的一辆辆具体汽车 → 每辆车颜色、里程可能不同，但结构相同
  • 实例化（Instantiation） = 按图纸造车的过程

一张设计图可以造出无数辆汽车，每辆相互独立、互不影响。这正是 OOP 的魔力。

📌 OOP 四大核心特性

🔒 封装（Encapsulation）

把数据和方法打包在一起，隐藏内部细节，只暴露必要接口。使用者无需了解实现，只需使用接口。

类比：汽车方向盘——你不需要知道转向系统内部构造，转动方向盘即可。

🧬 继承（Inheritance）

子类自动获得父类的属性和方法，实现代码复用。子类还可以扩展或覆盖父类行为。

类比：电动汽车继承了汽车的基本功能，还增加了充电等特有功能。

🎭 多态（Polymorphism）

不同类的对象可以响应相同的方法调用，但执行各自特有的行为。"同一接口，不同实现"。

类比：speak()方法——狗叫汪汪，猫叫喵喵，接口相同，行为不同。

🎨 抽象（Abstraction）

只暴露对象的关键信息，隐藏复杂实现。关注"是什么、能做什么"，而非"怎么实现"。

类比：ATM机——你只需要知道如何取钱，不需要了解内部电路原理。

📌 面向过程 vs 面向对象

维度面向过程（POP）面向对象（OOP）
核心单元函数/过程类/对象
数据与行为分离封装在一起
代码复用函数调用继承、组合、多态
程序结构顺序执行对象交互
适用场景简单、线性任务，脚本复杂系统，大型项目
优点简单直观，执行效率高可维护性强，扩展性好
缺点规模大时难以维护学习曲线较陡
Python 支持多范式：你完全可以在一个项目中混用面向过程和面向对象。小脚本用函数，复杂系统用类，根据场景选择最合适的方式！

🏗️ 2. 类与对象：核心概念深入

📌 定义第一个类

用 class 关键字定义类，用 类名() 创建对象。来看一个完整示例：

# 定义"学生"类
class Student:
    """学生类 —— 这是类的文档字符串（docstring），描述类的用途"""

    def __init__(self, name, age, score):
        self.name = name      # 实例属性：每个对象独立拥有
        self.age = age
        self.score = score

    def introduce(self):
        return f"大家好！我叫{self.name}，{self.age}岁，成绩{self.score}分。"

    def study(self, subject):
        return f"{self.name} 正在学习 {subject}..."

    def is_excellent(self):
        return self.score >= 90

# ——— 实例化（创建对象）———
s1 = Student("小明", 18, 92)
s2 = Student("小红", 17, 98)
s3 = Student("小刚", 19, 75)

# 访问属性
print(s1.name)    # 小明
print(s2.age)     # 17

# 调用方法
print(s1.introduce())       # 大家好！我叫小明，18岁，成绩92分。
print(s1.is_excellent())    # True
print(s3.is_excellent())    # False

# s1和s2是完全独立的对象
print(s1 is s2)                       # False
print(type(s1))                       # <class '__main__.Student'>
print(isinstance(s1, Student))       # True

📌 对象在内存中的存储原理

理解内存模型有助于避免很多坑：

# 变量名是对象的"引用"（指针），而非对象本身
s1 = Student("小明", 18, 92)
s4 = s1                    # s4 和 s1 指向【同一个对象】！
s4.name = "小强"           # 通过s4修改，s1也变了
print(s1.name)             # 小强（s1也变了！）
print(id(s1) == id(s4))   # True，是同一个内存地址

# 如何创建真正独立的副本？
import copy
s5 = copy.copy(s1)         # 浅拷贝：新对象，但嵌套对象仍共享
s6 = copy.deepcopy(s1)     # 深拷贝：完全独立的新对象（推荐）
s6.name = "完全独立"
print(s1.name)             # 小强（s6不影响s1）

# 查看对象的属性字典
print(s1.__dict__)         # {'name': '小强', 'age': 18, 'score': 92}
命名规范：类名用大驼峰（PascalCase），如 StudentInfo、BankAccount。方法和属性用小写加下划线，如 get_balance()、first_name。

🔧 3. __init__ 初始化方法与属性详解

🔑 __init__ 的本质原理

__init__ 不是"构造函数"，而是"初始化方法"。Python 创建对象的完整流程是：

  1. __new__ 方法：在内存中分配空间，创建空对象（通常不需要重写）
  2. __init__ 方法：对空对象进行属性初始化赋值
  3. 返回初始化完毕的对象引用

每次执行 Student("小明", 18, 92)，Python 都在堆内存中创建一块新的区域来存储这个对象。

📌 __init__ 参数设计最佳实践

class Person:
    """人员信息类"""

    def __init__(self, name, age=18, city="北京", hobbies=None):
        # ✅ 必填参数：name
        if not name or not isinstance(name, str):
            raise ValueError("姓名不能为空")
        self.name = name.strip()

        # ✅ 带默认值的可选参数
        self.age = age
        self.city = city

        # ✅ 可变对象用 None 作默认值（避免经典陷阱！）
        self.hobbies = hobbies if hobbies is not None else []

        # ✅ 内部私有属性（不在参数中暴露）
        self._id = id(self)        # 用内存地址作唯一ID
        self._created = "2026-04-02"

    def __repr__(self):
        return f"Person(name='{self.name}', age={self.age}, city='{self.city}')"

# 各种实例化方式
p1 = Person("张三")                      # 只传必填参数
p2 = Person("李四", 25, "上海")           # 位置参数
p3 = Person("王五", city="广州", age=30)  # 关键字参数（顺序可变）
p4 = Person("赵六", hobbies=["阅读", "编程"])
print(p1)   # Person(name='张三', age=18, city='北京')
print(p4.hobbies)  # ['阅读', '编程']
🚨 可变默认参数陷阱（Python常见错误）：
错误写法：def __init__(self, hobbies=[]): ← 所有实例共享同一个列表！
正确写法：def __init__(self, hobbies=None): self.hobbies = hobbies or []
因为默认参数在函数定义时就被创建，所有调用都共享同一个对象，这是Python初学者最容易踩的坑之一。

📌 初始化时进行数据验证

class BankAccount:
    """银行账户类 —— 演示完善的初始化验证"""

    def __init__(self, owner, balance=0.0, account_type="储蓄"):
        # 验证账户持有人
        if not isinstance(owner, str) or not owner.strip():
            raise TypeError("账户持有人必须是非空字符串")

        # 验证初始余额
        if not isinstance(balance, (int, float)):
            raise TypeError("余额必须是数字")
        if balance < 0:
            raise ValueError("初始余额不能为负数")

        # 验证账户类型
        valid_types = ["储蓄", "支票", "理财"]
        if account_type not in valid_types:
            raise ValueError(f"账户类型必须是：{valid_types}")

        self.owner = owner.strip()
        self._balance = float(balance)
        self.account_type = account_type
        self._history = []

    def deposit(self, amount):
        if amount <= 0:
            raise ValueError("存款金额必须大于0")
        self._balance += amount
        self._history.append(f"存入 ¥{amount:.2f}")
        return self      # 返回self支持链式调用

    def withdraw(self, amount):
        if amount <= 0:
            raise ValueError("取款金额必须大于0")
        if amount > self._balance:
            raise ValueError(f"余额不足，当前余额 ¥{self._balance:.2f}")
        self._balance -= amount
        self._history.append(f"取出 ¥{amount:.2f}")
        return self

    @property
    def balance(self):
        return self._balance

    def show_history(self):
        print(f"=== {self.owner} 的账单 ===")
        for record in self._history:
            print(f"  {record}")
        print(f"当前余额：¥{self._balance:.2f}")

# 链式调用
acc = BankAccount("周同学", 1000)
acc.deposit(500).deposit(200).withdraw(300)
acc.show_history()
# === 周同学 的账单 ===
#   存入 ¥500.00
#   存入 ¥200.00
#   取出 ¥300.00
# 当前余额：¥1400.00

📌 @property 属性装饰器

@property 让方法像属性一样访问，常用于计算属性和数据保护：

class Circle:
    def __init__(self, radius):
        self.radius = radius   # 触发 setter 验证

    @property
    def radius(self):
        return self._radius

    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("半径不能为负数")
        self._radius = value

    @property
    def area(self):             # 只读计算属性
        import math
        return math.pi * self._radius ** 2

    @property
    def circumference(self):    # 只读计算属性
        import math
        return 2 * math.pi * self._radius

c = Circle(5)
print(c.radius)              # 5  （像访问属性一样）
print(f"{c.area:.2f}")       # 78.54
print(f"{c.circumference:.2f}")  # 31.42
c.radius = 10                # 触发 setter，带验证
# c.radius = -1             # ValueError: 半径不能为负数
# c.area = 100              # AttributeError: 只读属性，不能赋值

⚡ 4. 实例方法与 self 深入解析

🔑 self 的本质

self 是对"当前实例"的引用，相当于其他语言中的 this。

当你调用 s1.introduce() 时，Python 实际执行的是 Student.introduce(s1)——把实例自动作为第一个参数传入。

self 是惯例命名（也可用其他名，但强烈建议保持 self），让方法知道"我在操作哪个对象的数据"。

📌 三种方法类型对比

class MethodDemo:
    class_counter = 0    # 类属性，所有实例共享

    def __init__(self, value):
        self.value = value
        MethodDemo.class_counter += 1

    # 1. 实例方法（最常用）—— 第一参数必须是 self
    def instance_method(self):
        # 可以访问实例属性 和 类属性
        return f"实例方法：value={self.value}, counter={self.class_counter}"

    # 2. 类方法 —— @classmethod，第一参数是 cls（类本身）
    @classmethod
    def class_method(cls):
        # 只能访问类属性，不能访问实例属性（无self）
        return f"类方法：已创建 {cls.class_counter} 个实例"

    # 3. 静态方法 —— @staticmethod，无 self/cls
    @staticmethod
    def static_method(x, y):
        # 不依赖实例或类，只是逻辑上归属于此类
        return x + y

obj1 = MethodDemo(10)
obj2 = MethodDemo(20)

print(obj1.instance_method())   # 实例方法：value=10, counter=2
print(MethodDemo.class_method()) # 类方法：已创建 2 个实例
print(MethodDemo.static_method(3, 4))  # 7

📌 类方法实战：工厂方法模式

类方法最常见的应用是"工厂方法"——从不同格式数据创建对象：

class Color:
    """颜色类，支持多种格式创建"""

    def __init__(self, r, g, b):
        self.r, self.g, self.b = r, g, b

    @classmethod
    def from_hex(cls, hex_str):
        """从十六进制字符串创建，如 '#FF8800'"""
        hex_str = hex_str.lstrip('#')
        r = int(hex_str[0:2], 16)
        g = int(hex_str[2:4], 16)
        b = int(hex_str[4:6], 16)
        return cls(r, g, b)

    @classmethod
    def from_tuple(cls, rgb_tuple):
        """从元组创建，如 (255, 136, 0)"""
        return cls(*rgb_tuple)

    @staticmethod
    def is_valid_rgb(r, g, b):
        """验证RGB值是否合法"""
        return all(0 <= v <= 255 for v in (r, g, b))

    def to_hex(self):
        return f"#{self.r:02X}{self.g:02X}{self.b:02X}"

    def __repr__(self):
        return f"Color(r={self.r}, g={self.g}, b={self.b})"

# 三种方式创建同一颜色
c1 = Color(255, 136, 0)
c2 = Color.from_hex("#FF8800")
c3 = Color.from_tuple((255, 136, 0))
print(c1, c2, c3)        # 三个完全一样
print(c1.to_hex())       # #FF8800
print(Color.is_valid_rgb(256, 0, 0))  # False
方法类型装饰器第一参数能访问实例属性能访问类属性常见用途
实例方法无self✅✅操作实例数据（最常用）
类方法@classmethodcls❌✅工厂方法、操作类状态
静态方法@staticmethod无❌❌工具函数、验证逻辑

📦 5. 类属性 vs 实例属性

🔑 两种属性的区别
  • 类属性：定义在类体中、方法外，所有实例共享同一个值，通过 类名.属性 或 self.属性 访问
  • 实例属性：在 __init__ 中用 self.属性 定义，每个实例独立拥有，互不影响
class Employee:
    # 类属性：所有员工共享
    company = "科技有限公司"
    employee_count = 0
    base_salary = 5000

    def __init__(self, name, department, salary_factor=1.0):
        # 实例属性：每位员工独立
        self.name = name
        self.department = department
        self.salary_factor = salary_factor
        self._salary = Employee.base_salary * salary_factor

        # 每创建一个员工，类属性计数+1
        Employee.employee_count += 1

    @property
    def salary(self):
        return self._salary

    @classmethod
    def raise_base_salary(cls, percent):
        """全员基础工资调整（修改类属性）"""
        cls.base_salary *= (1 + percent / 100)
        print(f"基础工资调整为 ¥{cls.base_salary:.0f}")

    def __repr__(self):
        return f"{self.name}({self.department}) ¥{self._salary:.0f}/月"

e1 = Employee("张三", "技术部", 1.5)
e2 = Employee("李四", "市场部", 1.2)
e3 = Employee("王五", "运营部")

# 通过类名访问类属性
print(Employee.company)         # 科技有限公司
print(Employee.employee_count)   # 3
print(e1, e2, e3)

# 通过实例访问类属性（也可以）
print(e1.company)               # 科技有限公司
注意：实例属性会"遮蔽"类属性
当你写 self.company = "新公司" 时，不是修改类属性，而是给该实例创建了一个同名的实例属性，遮蔽了类属性。其他实例的 company 不受影响。要修改所有实例共享的类属性，必须用 Employee.company = "新公司"。

📌 属性查找顺序（MRO）

Python 查找属性的顺序：实例字典 → 类字典 → 父类字典（按MRO顺序）

class A:
    x = "类属性"

a = A()
print(a.x)           # "类属性" —— 从类字典找到

a.x = "实例属性"      # 在实例字典中创建新属性
print(a.x)           # "实例属性" —— 实例字典优先
print(A.x)           # "类属性" —— 类属性不受影响

del a.x              # 删除实例属性
print(a.x)           # "类属性" —— 重新从类字典找

🔒 6. 封装：数据保护机制

🔑 封装的原理与意义

封装是将数据（属性）和操作数据的方法绑定在一起，同时限制外部对内部数据的直接访问，只通过受控的接口（方法）进行交互。

好的封装让类的使用者不需要关心内部实现，类的维护者也可以随时修改内部实现而不影响外部调用。

📌 Python 的三种访问级别

命名约定访问级别示例说明
name公开（Public）self.name任何地方都可访问，默认
_name受保护（Protected）self._balance约定"内部使用"，外部仍可访问，但不推荐
__name私有（Private）self.__secret名称改写（name mangling），外部难以直接访问
class SecureVault:
    """安全保险柜 —— 演示三种访问级别"""

    def __init__(self, owner, pin):
        self.owner = owner           # 公开属性
        self._contents = []          # 受保护（约定内部使用）
        self.__pin = pin             # 私有（Python会改写名称）
        self.__attempts = 0

    def deposit(self, item, pin):
        if not self.__verify_pin(pin):
            return "PIN错误，拒绝访问"
        self._contents.append(item)
        return f"已存入：{item}"

    def view(self, pin):
        if not self.__verify_pin(pin):
            return "PIN错误，拒绝查看"
        return f"保险柜内容：{self._contents}"

    def __verify_pin(self, pin):
        """私有方法：验证PIN码"""
        if self.__attempts >= 3:
            return "账户已锁定"
        if pin == self.__pin:
            self.__attempts = 0
            return True
        self.__attempts += 1
        return False

vault = SecureVault("周同学", "1234")
print(vault.deposit("黄金", "1234"))   # 已存入：黄金
print(vault.view("1234"))             # 保险柜内容：['黄金']
print(vault.deposit("现金", "0000"))   # PIN错误，拒绝访问

# 访问私有属性：Python会把 __pin 改写为 _SecureVault__pin
# print(vault.__pin)         # AttributeError!
print(vault._SecureVault__pin)         # "1234"（name mangling，不推荐这样用）

📌 封装实战：温度转换器

class Temperature:
    """温度类 —— 内部用摄氏度存储，支持多种单位"""

    def __init__(self, celsius=0):
        self.celsius = celsius   # 触发setter

    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("温度不能低于绝对零度 -273.15°C")
        self._celsius = value

    @property
    def fahrenheit(self):
        return self._celsius * 9/5 + 32

    @fahrenheit.setter
    def fahrenheit(self, value):
        self.celsius = (value - 32) * 5/9

    @property
    def kelvin(self):
        return self._celsius + 273.15

    def __repr__(self):
        return f"{self._celsius:.2f}°C / {self.fahrenheit:.2f}°F / {self.kelvin:.2f}K"

t = Temperature(100)
print(t)                  # 100.00°C / 212.00°F / 373.15K
t.fahrenheit = 32         # 通过华氏度设置
print(t)                  # 0.00°C / 32.00°F / 273.15K
# t.celsius = -300         # ValueError!

🧬 7. 继承：代码复用的利器

🔑 继承的原理

继承允许一个类（子类/派生类）自动获得另一个类（父类/基类）的所有属性和方法，在此基础上添加新功能或修改现有功能。

继承体现了"IS-A"关系：电动汽车 IS-A 汽车，学生 IS-A 人。

Python 中所有类都隐式继承自 object（根类），这就是为什么所有对象都有 __str__、__repr__ 等方法。

🎭 8. 多态与鸭子类型

🔑 多态的本质

多态（Polymorphism）= "多种形态"。同一个方法名，在不同的类中有不同的实现，使用时可以统一调用，不关心具体类型。

Python 的多态基于"鸭子类型（Duck Typing）"："如果它走路像鸭子，叫声像鸭子，那它就是鸭子。" —— Python 不看类型，只看行为（方法）。

📌 多态与抽象基类（ABC）

from abc import ABC, abstractmethod

class Shape(ABC):
    """抽象基类：不能直接实例化，规定了子类必须实现的接口"""

    @abstractmethod
    def area(self):
        """子类必须实现 area() 方法"""
        pass

    @abstractmethod
    def perimeter(self):
        """子类必须实现 perimeter() 方法"""
        pass

    def describe(self):
        # 非抽象方法可以有默认实现
        return f"{self.__class__.__name__}: 面积={self.area():.2f}, 周长={self.perimeter():.2f}"

import math

class Circle(Shape):
    def __init__(self, radius): self.radius = radius
    def area(self): return math.pi * self.radius ** 2
    def perimeter(self): return 2 * math.pi * self.radius

class Rectangle(Shape):
    def __init__(self, w, h): self.w, self.h = w, h
    def area(self): return self.w * self.h
    def perimeter(self): return 2 * (self.w + self.h)

class Triangle(Shape):
    def __init__(self, a, b, c): self.a, self.b, self.c = a, b, c
    def area(self):
        s = (self.a + self.b + self.c) / 2
        return math.sqrt(s*(s-self.a)*(s-self.b)*(s-self.c))
    def perimeter(self): return self.a + self.b + self.c

# 多态的威力：统一处理不同类型的形状
shapes = [Circle(5), Rectangle(4, 6), Triangle(3, 4, 5)]
for shape in shapes:
    print(shape.describe())
# Circle: 面积=78.54, 周长=31.42
# Rectangle: 面积=24.00, 周长=20.00
# Triangle: 面积=6.00, 周长=12.00

# 计算总面积（不关心具体类型！）
total_area = sum(shape.area() for shape in shapes)
print(f"总面积：{total_area:.2f}")

# 不能实例化抽象类
# Shape()  # TypeError: Can't instantiate abstract class Shape
鸭子类型的优势：Python 不强制要求继承关系。只要一个对象有 area() 和 perimeter() 方法，就可以当作 Shape 使用，无需显式声明继承。这让代码更加灵活。

✨ 9. 魔术方法（特殊方法/Dunder Methods）

🔑 什么是魔术方法

魔术方法（Magic Methods）是以双下划线开头和结尾的特殊方法，如 __init__、__str__、__add__。

📌 运算符重载：让类支持 + - * == 等操作

class Vector:
    """二维向量类 —— 演示运算符重载"""

    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        """v1 + v2"""
        return Vector(self.x + other.x, self.y + other.y)

    def __sub__(self, other):
        """v1 - v2"""
        return Vector(self.x - other.x, self.y - other.y)

    def __mul__(self, scalar):
        """v * 数字（标量乘法）"""
        return Vector(self.x * scalar, self.y * scalar)

    def __rmul__(self, scalar):
        """数字 * v（反向乘法，如 3 * v）"""
        return self.__mul__(scalar)

    def __eq__(self, other):
        """v1 == v2"""
        return isinstance(other, Vector) and self.x == other.x and self.y == other.y

    def __abs__(self):
        """abs(v)：向量模长"""
        import math
        return math.sqrt(self.x**2 + self.y**2)

    def __bool__(self):
        """非零向量为True"""
        return self.x != 0 or self.y != 0

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

v1 = Vector(1, 2)
v2 = Vector(3, 4)

print(v1 + v2)     # Vector(4, 6)
print(v2 - v1)     # Vector(2, 2)
print(v1 * 3)      # Vector(3, 6)
print(2 * v2)      # Vector(6, 8)
print(abs(v2))     # 5.0  (3-4-5三角形)
print(v1 == Vector(1, 2))  # True
print(bool(Vector(0, 0)))   # False

📌 上下文管理器：__enter__ 和 __exit__

class DatabaseConnection:
    """数据库连接 —— 演示上下文管理器"""

    def __init__(self, db_name):
        self.db_name = db_name
        self.connection = None

    def __enter__(self):
        """with块开始时调用，返回值赋给 as 后的变量"""
        print(f"连接数据库：{self.db_name}")
        self.connection = f"Connection({self.db_name})"
        return self  # 返回self，with ... as db 的db就是self

    def __exit__(self, exc_type, exc_val, exc_tb):
        """with块结束时调用（无论正常还是异常）"""
        print(f"关闭数据库连接：{self.db_name}")
        self.connection = None
        if exc_type is not None:
            print(f"捕获到异常：{exc_type.__name__}: {exc_val}")
            return True  # 返回True表示异常已处理，不继续传播

    def query(self, sql):
        if not self.connection:
            raise RuntimeError("未连接数据库")
        return f"执行查询：{sql}"

# with 语句自动管理连接的开启和关闭
with DatabaseConnection("mydb") as db:
    print(db.query("SELECT * FROM users"))
# 输出：
# 连接数据库：mydb
# 执行查询：SELECT * FROM users
# 关闭数据库连接：mydb

📌 可迭代对象：__iter__ 和 __next__

class Countdown:
    """倒计时迭代器"""

    def __init__(self, start):
        self.start = start

    def __iter__(self):
        """返回迭代器对象（这里self本身就是迭代器）"""
        self.current = self.start
        return self

    def __next__(self):
        """返回下一个值，没有更多值时抛出StopIteration"""
        if self.current < 0:
            raise StopIteration
        value = self.current
        self.current -= 1
        return value

for n in Countdown(5):
    print(n, end=" ")   # 5 4 3 2 1 0

# 也可以直接转list
print(list(Countdown(3)))  # [3, 2, 1, 0]

from datetime import datetime, timedelta
from abc import ABC, abstractmethod

# ============================================================
# 第一部分：书籍类（封装 + 继承 + 多态）
# ============================================================

class BaseBook(ABC):
    """书籍抽象基类"""
    _total_books = 0   # 类属性：统计总书目数

    def __init__(self, title, author, isbn, category):
        if not all([title, author, isbn]):
            raise ValueError("书名、作者、ISBN不能为空")
        self._title = title
        self._author = author
        self._isbn = isbn
        self.category = category
        self._borrow_count = 0
        self._available = True
        BaseBook._total_books += 1

    @property
    def title(self): return self._title

    @property
    def author(self): return self._author

    @property
    def isbn(self): return self._isbn

    @property
    def available(self): return self._available

    @property
    def borrow_count(self): return self._borrow_count

    @abstractmethod
    def get_type(self):
        """子类必须实现：返回书籍类型字符串"""
        pass

    @abstractmethod
    def get_borrow_days(self):
        """子类必须实现：返回可借天数"""
        pass

    def borrow(self):
        if not self._available:
            raise RuntimeError(f"《{self._title}》已被借出")
        self._available = False
        self._borrow_count += 1
        due = datetime.now() + timedelta(days=self.get_borrow_days())
        return due

    def return_book(self):
        if self._available:
            raise RuntimeError(f"《{self._title}》未被借出")
        self._available = True

    @classmethod
    def get_total_books(cls):
        return cls._total_books

    def __repr__(self):
        status = "可借" if self._available else "已借出"
        return f"[{self.get_type()}] 《{self._title}》 - {self._author} [{status}]"

    def __eq__(self, other):
        return isinstance(other, BaseBook) and self._isbn == other._isbn

    def __hash__(self):
        return hash(self._isbn)

class Member:
    """图书馆会员"""
    MAX_BORROW = 5   # 最多同时借5本

    def __init__(self, member_id, name):
        self.member_id = member_id
        self.name = name
        self._borrowed = {}   # {book: due_date}
        self._history = []    # 借阅历史

    def borrow_book(self, book):
        if len(self._borrowed) >= self.MAX_BORROW:
            raise RuntimeError(f"{self.name} 已达到最大借阅数 {self.MAX_BORROW} 本")
        due = book.borrow()
        self._borrowed[book] = due
        self._history.append(("借出", book.title, datetime.now()))
        return f"{self.name} 借出《{book.title}》，应还日期：{due.strftime('%Y-%m-%d')}"

    def return_book(self, book):
        if book not in self._borrowed:
            raise RuntimeError(f"{self.name} 没有借阅《{book.title}》")
        due = self._borrowed.pop(book)
        book.return_book()
        self._history.append(("归还", book.title, datetime.now()))
        overdue = (datetime.now() - due).days
        msg = f"{self.name} 归还《{book.title}》"
        if overdue > 0:
            msg += f" ⚠️ 超期 {overdue} 天，罚款 ¥{overdue * 0.5:.1f}"
        return msg

    @property
    def borrowed_count(self): return len(self._borrowed)

    def show_borrowed(self):
        if not self._borrowed:
            print(f"{self.name} 当前没有借阅书籍")
            return
        print(f"\n{self.name} 当前借阅（{len(self._borrowed)}/{self.MAX_BORROW}）：")
        for book, due in self._borrowed.items():
            print(f"  《{book.title}》 应还：{due.strftime('%Y-%m-%d')}")

    def __repr__(self):
        return f"Member({self.member_id}, '{self.name}', 已借{self.borrowed_count}本)"

class Library:
    """图书馆管理类"""

    def __init__(self, name):
        self.name = name
        self._books = {}    # {isbn: book}
        self._members = {}  # {member_id: member}

    def add_book(self, book):
        if book.isbn in self._books:
            raise ValueError(f"ISBN {book.isbn} 已存在")
        self._books[book.isbn] = book
        return self   # 支持链式调用

    def add_member(self, member):
        self._members[member.member_id] = member
        return self

    def search(self, keyword, field="title"):
        """按 title/author/category 搜索"""
        keyword = keyword.lower()
        results = []
        for book in self._books.values():
            target = getattr(book, field, "").lower()
            if keyword in target:
                results.append(book)
        return results

    def top_borrowed(self, n=5):
        """借阅量排行榜 Top N"""
        return sorted(self._books.values(),
                       key=lambda b: b.borrow_count, reverse=True)[:n]

    def stats(self):
        """统计信息"""
        total = len(self._books)
        available = sum(1 for b in self._books.values() if b.available)
        print(f"\n=== {self.name} 统计 ===")
        print(f"  总藏书：{total} 本")
        print(f"  可借阅：{available} 本")
        print(f"  借出中：{total - available} 本")
        print(f"  会员数：{len(self._members)} 人")

    def __len__(self):
        return len(self._books)

    def __contains__(self, isbn):
        return isbn in self._books

    def __repr__(self):
        return f"Library('{self.name}', {len(self._books)}本)"
