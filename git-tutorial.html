1. Git简介

什么是Git？

Git是一个分布式版本控制系统，用于跟踪文件和目录的变化，方便团队协作开发。最早由Linux之父Linus Torvalds创建。

核心概念

  • 仓库（Repository）：存放项目文件的地方
  • 提交（Commit）：保存文件的某个状态
  • 分支（Branch）：从主线分离出来的工作线
  • 合并（Merge）：将不同分支的内容合并
  • 远程仓库（Remote）：托管在服务器上的仓库

工作流程

工作目录(Working Directory)
        ↓ git add
暂存区(Staging Area)
        ↓ git commit
本地仓库(Local Repository)
        ↓ git push
远程仓库(Remote Repository)

2. 安装与配置

Windows安装

  1. 访问 Git官网
  2. 下载Windows版本的Git
  3. 运行安装程序，一路Next即可
  4. 安装完成后，右键菜单会有Git Bash Here选项

macOS安装

# 方式1：使用Homebrew
brew install git

# 方式2：安装Xcode命令行工具
xcode-select --install

Linux安装

# Ubuntu/Debian
sudo apt update
sudo apt install git

# CentOS/RHEL
sudo yum install git

基础配置

# 设置用户名（必填）
git config --global user.name "你的名字"

# 设置邮箱（必填）
git config --global user.email "your@email.com"

# 设置默认编辑器
git config --global core.editor vscode

# 查看配置
git config --list
💡 提示：每次提交都需要用户名和邮箱，这些信息会出现在你的提交记录中。

3. 基础操作

创建仓库

# 在当前目录创建仓库
git init

# 克隆远程仓库
git clone https://github.com/username/repo.git

查看状态

# 查看工作区状态
git status

# 简化输出
git status -s

添加文件

# 添加单个文件
git add filename.py

# 添加所有文件
git add .

# 添加所有.py文件
git add *.py

# 交互式添加
git add -i

提交

# 提交并写消息
git commit -m "提交说明"

# 提交已跟踪的文件（跳过git add）
git commit -am "提交说明"

# 修改最后一次提交（还没push时）
git commit --amend -m "新的提交说明"

# 查看提交历史
git log

# 简洁模式
git log --oneline

撤销操作

# 取消暂存（从暂存区移回工作区）
git reset HEAD filename.py

# 取消所有暂存
git reset HEAD

# 丢弃工作区的修改（慎用！）
git checkout -- filename.py

# 或者（新语法）
git restore filename.py

# 删除文件并提交
git rm filename.py
git commit -m "删除文件"
⚠️ 注意：git checkout会丢弃未提交的修改，请确保已经保存了重要更改。

查看差异

# 查看工作区vs暂存区的差异
git diff

# 查看暂存区vs仓库的差异
git diff --staged

# 查看两个分支的差异
git diff branch1..branch2

# 查看某个文件的变化
git diff filename.py

4. 分支管理

创建和切换分支

# 查看所有分支
git branch

# 查看远程分支
git branch -r

# 创建新分支
git branch feature-login

# 切换分支
git checkout feature-login

# 创建并切换（新语法）
git switch -c feature-login

# 切换到主分支
git checkout main
# 或
git switch main

合并分支

# 1. 切换到要合并到的分支（通常是main）
git checkout main

# 2. 合并指定分支
git merge feature-login

# 3. 删除已合并的分支
git branch -d feature-login

解决冲突

当两个分支修改了同一文件的同一位置时，会产生冲突：

<<<<<<< HEAD
当前分支的内容
=======
要合并的分支的内容
>>>>>>> feature-branch

解决方法：手动编辑文件，保留想要的内容，然后：

git add filename.py
git commit -m "解决冲突"

储藏工作

# 储藏当前工作
git stash

# 查看储藏列表
git stash list

# 恢复储藏
git stash apply

# 恢复并删除
git stash pop

# 删除储藏
git stash drop

5. 远程仓库

添加远程仓库

# 添加远程仓库
git remote add origin https://github.com/username/repo.git

# 查看远程仓库
git remote -v

# 重命名远程仓库
git remote rename origin upstream

推送和拉取

# 推送到远程仓库（首次推送）
git push -u origin main

# 推送（之后可以省略-u）
git push

# 拉取远程更新
git pull

# 获取但不合并
git fetch

SSH密钥配置

  1. 生成SSH密钥： 
    ssh-keygen -t ed25519 -C "your@email.com"
  2. 查看公钥： 
    cat ~/.ssh/id_ed25519.pub
  3. 复制公钥到GitHub：Settings → SSH and GPG keys → New SSH key
  4. 测试连接： 
    ssh -T git@github.com

6. GitHub使用

创建仓库

  1. 登录GitHub，点击右上角 + → New repository
  2. 填写仓库名称、描述
  3. 选择Public或Private
  4. 点击Create repository

推送本地代码到GitHub

# 1. 初始化本地仓库
git init

# 2. 添加文件
git add .
git commit -m "Initial commit"

# 3. 添加远程仓库
git remote add origin https://github.com/username/repo.git

# 4. 推送
git push -u origin main

GitHub Flow工作流

# 1. 从远程拉取最新代码
git checkout main
git pull origin main

# 2. 创建新分支
git checkout -b feature/new-feature

# 3. 开发并提交
git add .
git commit -m "Add new feature"

# 4. 推送到远程
git push origin feature/new-feature

# 5. 在GitHub上创建Pull Request
# 6. 代码审查后合并到main分支

常用GitHub操作

操作 命令
克隆仓库 git clone URL
更新Fork的仓库 git remote add upstream URL
git pull upstream main
删除远程分支 git push origin --delete branch-name
强制推送 git push -f（慎用！）

7. 实用技巧

别名设置

# 设置简短别名
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit

# 高级别名
git config --global alias.lg "log --oneline --graph --all"

忽略文件

创建 .gitignore 文件：

# Python
__pycache__/
*.py[cod]
*.egg-info/
dist/
build/

# Virtual Environment
venv/
env/

# IDE
.vscode/
.idea/

# OS
.DS_Store
Thumbs.db

清理和优化

# 删除已合并的分支
git branch --merged main | grep -v "main" | xargs git branch -d

# 垃圾回收
git gc

# 检查仓库问题
git fsck

查看修改者

# 查看某行代码的最后修改者
git blame filename.py

# 查看最近一周的提交
git log --since="1 week ago"

# 查看某个作者的提交
git log --author="Author Name"

标签管理

# 创建标签
git tag v1.0.0

# 推送标签
git push origin v1.0.0

# 推送所有标签
git push origin --tags

# 删除标签
git tag -d v1.0.0
git push origin --delete v1.0.0
