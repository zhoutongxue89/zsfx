1️⃣ 并发 vs 并行：核心概念

🔑 核心定义
  • 并发（Concurrency）：同一个时间段内多个任务交替执行，CPU在多个任务间快速切换
  • 并行（Parallelism）：多个任务真正同时执行，需要多核CPU

📌 生活中的类比

🍳 一个人做早餐（并发）

先烧水，等水开的间隙去烤面包，烤面包时去煮咖啡...

看似同时做了多件事，实际上是交替执行

→ 单核CPU执行多线程

👨‍🍳 三个人同时做早餐（并行）

一人烧水，一人烤面包，一人煮咖啡

真正的同时执行

→ 多核CPU执行多线程

📌 Python中的选择

场景 推荐方案 原因
I/O密集型（网络请求、文件读写） 多线程 / asyncio 等待时CPU可处理其他任务
CPU密集型（计算、图像处理） 多进程 绕过GIL，充分利用多核
I/O等待时间长 asyncio 更轻量，无需线程切换开销
💡 提示 Python中约80%的任务属于I/O密集型（网络请求、文件操作、数据库查询等），多线程在这些场景下非常有效！

2️⃣ GIL全局解释器锁

⚠️ 重要概念

GIL（Global Interpreter Lock）是Python最重要的性能特性之一，理解它对于选择并发方案至关重要！

📌 什么是GIL？

GIL是Python解释器（CPython）中的一个互斥锁，它确保任意时刻只有一个线程执行Python字节码。

📌 GIL的影响

❌ CPU密集型任务

import time

def cpu_task():
    total = 0
    for i in range(10_000_000):
        total += i
    return total

# 单线程
start = time.time()
cpu_task()
print(f"单线程: {time.time() - start:.2f}s")

# 多线程（没有加速！）
import threading
start = time.time()
t1 = threading.Thread(target=cpu_task)
t2 = threading.Thread(target=cpu_task)
t1.start(); t2.start()
t1.join(); t2.join()
print(f"多线程: {time.time() - start:.2f}s")  # 可能更慢！

结果：多线程可能比单线程更慢，因为线程切换也有开销

✅ I/O密集型任务

import time
import requests

def fetch_url(url):
    response = requests.get(url)
    return len(response.content)

urls = ['https://example.com'] * 10

# 单线程（串行执行）
start = time.time()
for url in urls:
    fetch_url(url)
print(f"单线程: {time.time() - start:.2f}s")

# 多线程（并发执行）
from concurrent.futures import ThreadPoolExecutor
start = time.time()
with ThreadPoolExecutor(max_workers=5) as executor:
    results = list(executor.map(fetch_url, urls))
print(f"多线程: {time.time() - start:.2f}s")  # 显著加速！

结果：多线程大幅加速，因为等待I/O时可以切换

✅ 记住这个原则

多线程适合I/O密集型任务，多进程适合CPU密集型任务

4️⃣ 线程的创建与启动

📌 方法一：直接创建线程函数

import threading
import time

def task(name, seconds):
    print(f"线程 {name} 开始")
    time.sleep(seconds)
    print(f"线程 {name} 完成")

# 创建线程
t1 = threading.Thread(target=task, args=("A", 2))
t2 = threading.Thread(target=task, args=("B", 1))

# 启动线程
t1.start()
t2.start()

# 等待线程完成
t1.join()
t2.join()

print("所有线程完成")
💡 运行结果

线程 B 开始（因为sleep时间更短）
线程 A 开始
线程 B 完成
线程 A 完成
所有线程完成

📌 方法二：继承Thread类

import threading
import time

class MyThread(threading.Thread):
    def __init__(self, name, seconds):
        super().__init__()
        self.name = name
        self.seconds = seconds
    
    def run(self):  # 线程执行的入口
        print(f"线程 {self.name} 开始")
        time.sleep(self.seconds)
        print(f"线程 {self.name} 完成")

# 使用
threads = []
for i in range(3):
    t = MyThread(f"Worker-{i}", 2-i)
    threads.append(t)
    t.start()

# 等待所有线程
for t in threads:
    t.join()

print("全部完成")

📌 方法三：守护线程

import threading
import time

def background_task():
    """后台任务 - 守护线程"""
    while True:
        print("后台监控中...")
        time.sleep(2)

# 创建守护线程
daemon = threading.Thread(target=background_task, daemon=True)
daemon.start()

# 主线程工作
print("主线程开始工作...")
time.sleep(5)
print("主线程完成，程序退出")
# 注意：主线程结束后，守护线程也会自动结束
⚠️ 注意事项
  • 守护线程(daemon=True)会在主线程结束时被强制终止
  • 适用于日志记录、健康检查等后台任务
  • 需要优雅关闭的任务不要用守护线程

5️⃣ 线程同步与锁

🔑 为什么要锁？

多个线程同时访问共享资源时，可能导致数据竞争（Race Condition），造成数据不一致。

📌 数据竞争示例

import threading

counter = 0

def increment():
    global counter
    for _ in range(1000000):
        counter += 1  # 这不是原子操作！

# 两个线程各加100万次
t1 = threading.Thread(target=increment)
t2 = threading.Thread(target=increment)

t1.start(); t2.start()
t1.join(); t2.join()

print(f"计数器结果: {counter}")
# 期望: 2000000
# 实际: 可能小于2000000！
⚠️ 问题原因

counter += 1实际上是三步操作：

  1. 读取counter的值
  2. 加1
  3. 写回counter

线程A在线程B还没写回时就读取了，导致数据丢失！

📌 方法一：Lock（互斥锁）

import threading

counter = 0
lock = threading.Lock()  # 创建锁

def increment():
    global counter
    for _ in range(1000000):
        with lock:  # 获取锁
            counter += 1  # 安全操作

t1 = threading.Thread(target=increment)
t2 = threading.Thread(target=increment)

t1.start(); t2.start()
t1.join(); t2.join()

print(f"计数器结果: {counter}")  # 正确: 2000000
💡 with语句自动管理锁

使用with lock:会在代码块结束时自动释放锁，即使发生异常也会释放，比手动lock/unlock更安全。

📌 方法二：RLock（可重入锁）

import threading

lock = threading.RLock()

def outer():
    with lock:
        print("外层获取锁")
        inner()  # 同一个线程再次获取锁
        print("外层继续")

def inner():
    with lock:  # 同一线程可以再次获取
        print("内层获取锁")

# RLock允许同一线程多次获取锁
outer()

📌 方法三：其他同步原语

类型 用途 示例
Lock 基本的互斥锁 保护共享数据
RLock 可重入锁 递归调用场景
Semaphore 信号量，限制并发数 连接池、限流
Event 事件触发 线程间通知
Condition 条件变量 生产者-消费者

📌 Semaphore示例：连接池

import threading
import time

class ConnectionPool:
    def __init__(self, size):
        self.size = size
        self.semaphore = threading.Semaphore(size)
        self.connections = []
    
    def acquire(self):
        self.semaphore.acquire()
        conn = f"连接-{id(self)}"
        self.connections.append(conn)
        return conn
    
    def release(self, conn):
        self.connections.remove(conn)
        self.semaphore.release()

pool = ConnectionPool(3)

def worker(name):
    conn = pool.acquire()
    print(f"{name} 获取 {conn}")
    time.sleep(1)
    print(f"{name} 释放 {conn}")
    pool.release(conn)

# 同时只有3个线程能获取连接
threads = [threading.Thread(target=worker, args=(f"Worker-{i}",)) for i in range(6)]
for t in threads: t.start()
for t in threads: t.join()

6️⃣ 死锁与解决方案

⚠️ 死锁是并发编程中最危险的问题之一

死锁发生时，程序会完全卡住，无法继续执行也无法退出。

📌 死锁的四个必要条件

  1. 互斥：资源只能被一个线程持有
  2. 持有并等待：线程持有资源的同时请求其他资源
  3. 不可抢占：资源不能被强制释放
  4. 循环等待：形成线程间的循环等待链

破坏任意一个条件即可避免死锁。

📌 经典死锁示例

import threading

lock_a = threading.Lock()
lock_b = threading.Lock()

def task1():
    with lock_a:
        print("任务1: 获取锁A")
        time.sleep(0.1)  # 模拟处理
        with lock_b:  # 等待锁B
            print("任务1: 获取锁B")
            print("任务1: 完成")

def task2():
    with lock_b:
        print("任务2: 获取锁B")
        time.sleep(0.1)
        with lock_a:  # 等待锁A
            print("任务2: 获取锁A")
            print("任务2: 完成")

# 死锁！
t1 = threading.Thread(target=task1)
t2 = threading.Thread(target=task2)
t1.start(); t2.start()
# 程序会卡住...

📌 解决方案一：固定加锁顺序

import threading

lock_a = threading.Lock()
lock_b = threading.Lock()

def task1_fixed():
    with lock_a:
        print("任务1: 获取锁A")
        time.sleep(0.1)
        with lock_b:  # 保持与task2相同的顺序
            print("任务1: 获取锁B")
            print("任务1: 完成")

def task2_fixed():
    with lock_a:  # 固定顺序：总是先A后B
        print("任务2: 获取锁A")
        time.sleep(0.1)
        with lock_b:
            print("任务2: 获取锁B")
            print("任务2: 完成")

# 不会死锁
t1 = threading.Thread(target=task1_fixed)
t2 = threading.Thread(target=task2_fixed)
t1.start(); t2.start()

📌 解决方案二：try_lock（超时获取）

import threading

lock_a = threading.Lock()
lock_b = threading.Lock()

def task1_try():
    while True:
        if lock_a.acquire(timeout=1):  # 尝试获取A
            print("任务1: 获取锁A")
            if lock_b.acquire(timeout=1):  # 尝试获取B
                print("任务1: 获取锁B")
                print("任务1: 完成")
                lock_b.release()
                lock_a.release()
                break
            else:
                lock_a.release()  # 获取B失败，释放A重试
                time.sleep(0.1)
        else:
            time.sleep(0.1)

def task2_try():
    while True:
        if lock_a.acquire(timeout=1):
            print("任务2: 获取锁A")
            if lock_b.acquire(timeout=1):
                print("任务2: 获取锁B")
                print("任务2: 完成")
                lock_b.release()
                lock_a.release()
                break
            else:
                lock_a.release()
                time.sleep(0.1)
        else:
            time.sleep(0.1)

📌 解决方案三：LockOrderWarning

Python 3.10+ 引入了调试工具来检测锁顺序问题：

# 设置环境变量启用死锁检测
import os
os.environ['PYTHONPHUBACKUP'] = '1'
# 或使用命令行

✅ 最佳实践
  • 尽量减少锁的使用范围
  • 多个锁时保持固定的获取顺序
  • 使用超时避免永久等待
  • 考虑使用高级抽象（队列、线程池）

📌 Condition条件变量

import threading
import time

class Buffer:
    def __init__(self, maxsize=10):
        self.buffer = []
        self.maxsize = maxsize
        self.condition = threading.Condition()
    
    def produce(self, item):
        with self.condition:
            while len(self.buffer) >= self.maxsize:
                self.condition.wait()  # 缓冲区满，等待
            self.buffer.append(item)
            print(f"生产: {item}")
            self.condition.notify()  # 通知消费者
    
    def consume(self):
        with self.condition:
            while not self.buffer:
                self.condition.wait()  # 缓冲区空，等待
            item = self.buffer.pop(0)
            print(f"消费: {item}")
            self.condition.notify()  # 通知生产者
            return item

buffer = Buffer()

def producer():
    for i in range(5):
        buffer.produce(i)
        time.sleep(0.5)

def consumer():
    for _ in range(5):
        item = buffer.consume()
        time.sleep(1)

t1 = threading.Thread(target=producer)
t2 = threading.Thread(target=consumer)
t1.start(); t2.start()

📌 Queue队列（推荐方式）

import threading
import queue
import time

# 创建线程安全的队列
q = queue.Queue(maxsize=5)

def producer():
    for i in range(10):
        q.put(i)  # 阻塞直到队列不满
        print(f"生产: {i}")
        time.sleep(0.3)

def consumer():
    while True:
        item = q.get()  # 阻塞直到队列不空
        print(f"消费: {item}")
        q.task_done()  # 标记完成
        time.sleep(0.5)
        if item == 9:  # 最后一项
            break

# 启动生产者
t1 = threading.Thread(target=producer)
# 启动消费者
t2 = threading.Thread(target=consumer)

t1.start()
t2.start()
t1.join()
t2.join()
print("完成")
💡 Queue的优势
  • 线程安全，无需手动加锁
  • 支持阻塞操作（put/get）
  • 内置大小限制
  • 支持join等待所有任务完成

8️⃣ 线程池与concurrent.futures

🔑 为什么需要线程池？

创建/销毁线程有开销，线程池复用线程，避免频繁创建销毁的开销。

📌 ThreadPoolExecutor基本用法

from concurrent.futures import ThreadPoolExecutor, as_completed
import time

def task(n):
    time.sleep(1)
    return n * n

# 创建线程池（5个工作线程）
with ThreadPoolExecutor(max_workers=5) as executor:
    # 提交多个任务
    futures = [executor.submit(task, i) for i in range(10)]
    
    # 等待所有任务完成并获取结果
    for future in as_completed(futures):
        result = future.result()
        print(f"结果: {result}")

📌 map方法（更简洁）

from concurrent.futures import ThreadPoolExecutor

def fetch_data(url):
    return f"数据: {url}"

urls = [f"http://api.com/item/{i}" for i in range(20)]

with ThreadPoolExecutor(max_workers=10) as executor:
    # 自动分配任务，结果按提交顺序返回
    results = executor.map(fetch_data, urls)
    
    for result in results:
        print(result)

📌 回调函数

from concurrent.futures import ThreadPoolExecutor

def download(url):
    return f"下载完成: {url}"

def process_result(future):
    print(f"回调处理: {future.result()}")

with ThreadPoolExecutor(max_workers=3) as executor:
    for url in ["file1.txt", "file2.txt", "file3.txt"]:
        future = executor.submit(download, url)
        future.add_done_callback(process_result)  # 完成后自动调用

📌 线程池大小选择

任务类型 推荐线程数 原因
I/O密集型 CPU核心数 × 2~4 I/O等待时CPU空闲，可多一些
CPU密集型 CPU核心数 + 1 避免过度上下文切换
网络爬虫 50-100（I/O等待长） 大量等待网络响应
数据库连接池 连接池大小 不超过数据库最大连接数

9️⃣ 实战：多线程下载器

🔑 项目目标

使用多线程实现一个文件下载器，支持并发下载多个文件。

import threading
import queue
import time
import os
from urllib.request import urlopen
from urllib.error import URLError

class MultiThreadDownloader:
    def __init__(self, max_workers=5):
        self.queue = queue.Queue()
        self.max_workers = max_workers
        self.results = []
        self.lock = threading.Lock()
    
    def download(self, url, filename):
        """下载单个文件"""
        try:
            print(f"[下载] {url} -> {filename}")
            
            with urlopen(url, timeout=30) as response:
                content = response.read()
            
            with open(filename, 'wb') as f:
                f.write(content)
            
            result = {"status": "success", "url": url, "file": filename}
            print(f"[完成] {filename} ({len(content)} bytes)")
            
        except Exception as e:
            result = {"status": "error", "url": url, "error": str(e)}
            print(f"[失败] {url}: {e}")
        
        with self.lock:
            self.results.append(result)
    
    def worker(self):
        """工作线程"""
        while True:
            try:
                url, filename = self.queue.get(timeout=1)
                self.download(url, filename)
            except queue.Empty:
                break
            finally:
                self.queue.task_done()
    
    def download_all(self, urls):
        """并发下载所有URL"""
        # 添加任务到队列
        for url in urls:
            filename = os.path.basename(url)
            if not filename:
                filename = f"download_{hash(url)}"
            self.queue.put((url, filename))
        
        # 启动工作线程
        threads = []
        for _ in range(self.max_workers):
            t = threading.Thread(target=self.worker)
            t.start()
            threads.append(t)
        
        # 等待所有任务完成
        self.queue.join()
        
        # 停止线程
        for t in threads:
            t.join()
        
        return self.results

📌 进阶：带进度条的下载器

import threading
import queue
from concurrent.futures import ThreadPoolExecutor, as_completed
from urllib.request import urlopen

class ProgressDownloader:
    def __init__(self, max_workers=3):
        self.max_workers = max_workers
        self.lock = threading.Lock()
        self.completed = 0
        self.total = 0
    
    def download_with_progress(self, url, filename):
        self.total += 1
        
        try:
            with urlopen(url, timeout=30) as resp:
                total_size = int(resp.headers.get('Content-Length', 0))
                downloaded = 0
                chunk_size = 8192
                
                with open(filename, 'wb') as f:
                    while True:
                        chunk = resp.read(chunk_size)
                        if not chunk:
                            break
                        f.write(chunk)
                        downloaded += len(chunk)
                        
                        if total_size:
                            progress = downloaded / total_size * 100
                            print(f"\r{filename}: {progress:.1f}%", end='', flush=True)
        except Exception as e:
            print(f"\n下载失败: {url} - {e}")
        finally:
            with self.lock:
                self.completed += 1
                print(f"\n进度: {self.completed}/{self.total}")
    
    def download_all(self, urls):
        with ThreadPoolExecutor(max_workers=self.max_workers) as executor:
            futures = {}
            for url in urls:
                filename = url.split('/')[-1]
                future = executor.submit(self.download_with_progress, url, filename)
                futures[future] = url
            
            # 等待完成
            for future in as_completed(futures):
                pass
        
        print(f"\n全部完成! 共 {self.total} 个文件")
