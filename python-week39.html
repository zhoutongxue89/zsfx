1️⃣ Docker简介

Docker是容器化平台，将应用和依赖打包到容器中，实现"一次构建，到处运行"。

✅ Docker优势

  • 环境一致
  • 快速部署
  • 资源隔离
  • 版本管理
  • 微服务友好

📌 核心概念

  • 镜像(Image): 只读模板
  • 容器(Container): 运行实例
  • 仓库(Registry): 镜像存储
  • Dockerfile: 构建脚本

2️⃣ Docker基础命令

# 镜像
docker pull python:3.12          # 拉取镜像
docker images                     # 列出镜像
docker rmi python:3.12            # 删除镜像

# 容器
docker run -d --name myapp -p 5000:5000 python:3.12  # 运行容器
docker ps                         # 运行中的容器
docker ps -a                      # 所有容器
docker stop myapp                 # 停止
docker start myapp                # 启动
docker rm myapp                   # 删除容器
docker logs myapp                 # 查看日志
docker exec -it myapp bash        # 进入容器

3️⃣ Dockerfile

# Dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["python", "app.py"]

# 构建镜像
docker build -t myapp:1.0 .

# 运行
docker run -d --name myapp -p 5000:5000 myapp:1.0

4️⃣ Docker Compose

# docker-compose.yml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "5000:5000"
    depends_on:
      - db
      - redis
    environment:
      - DATABASE_URL=mysql://user:pwd@db/mydb
      - REDIS_URL=redis://redis:6379
  
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: pwd
      MYSQL_DATABASE: mydb
    volumes:
      - db_data:/var/lib/mysql
  
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

volumes:
  db_data:

# 启动
docker-compose up -d
docker-compose logs -f
docker-compose down

6️⃣ Nginx部署

# nginx.conf
server {
    listen 80;
    server_name example.com;
    
    location / {
        proxy_pass http://web:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /static/ {
        alias /app/static/;
    }
}

# docker-compose添加nginx
nginx:
    image: nginx:alpine
    ports:
        - "80:80"
    volumes:
        - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
        - web
